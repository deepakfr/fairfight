üèóÔ∏è Architecture

User Flow

1
Initial Submission
User 1 submits their version of the conflict along with contact information for both parties.
2
Link Generation
System generates a secure token-based link and provides Email/WhatsApp sharing options.
3
User 2 Response
User 2 accesses the link, reviews User 1's version, and submits their perspective.
4
AI Analysis
JudgeBot analyzes both perspectives using GPT-4 and generates an impartial verdict.
5
Verdict Delivery
Both parties receive the verdict with win percentages and audio playback capability.

Data Storage Architecture

The application employs two append-only JSONL files for data persistence:

  • pending_cases.jsonl - Stores incomplete cases awaiting User 2's response
  • verdicts.jsonl - Permanent log of all delivered verdicts with full metadata

ü§ñ JudgeBot AI System

System Instruction

JudgeBot operates with the following core directive:

"You are JudgeBot, an impartial AI judge. Analyze both sides carefully, highlight key arguments from each, and give a fair verdict. Clearly state who is more reasonable, and give a win percentage (e.g., 60% vs 40%). You should give the response in the user texted language."

Language Processing Pipeline

  1. Detection: Uses langdetect on combined user input
  2. Translation: System instructions translated to detected language
  3. Response: AI responds in user's native language
  4. Speech Synthesis: TTS generated in detected language

üöÄ Deployment Guide

Local Development

# Configure secrets
mkdir .streamlit
echo '[openai]\napi_key = "sk-..."' > .streamlit/secrets.toml

# Run application

Streamlit Cloud Deployment

  1. Push your code to a GitHub repository
  2. Connect repository to Streamlit Cloud
  3. Add OpenAI API key in Secrets section
  4. Update BASE_URL constant to your deployment URL
  5. Deploy application
‚ö†Ô∏è File Permissions: Ensure the application has write access to pending_cases.jsonl and verdicts.jsonl. If file system access is unavailable, the fallback URL method will still function.

‚ö†Ô∏è Security Considerations

  • API Key Protection: Always store OpenAI keys in Streamlit secrets, never in code
  • Data Privacy: No authentication system; links serve as bearer tokens
  • Input Validation: Basic validation implemented on required fields
  • Rate Limiting: Not implemented; consider adding for production environments

üéØ Limitations

  • No user authentication or access control
  • Links do not expire automatically
  • No edit or delete functionality for submitted cases
  • Single AI model (GPT-4) with no customization options
  • File-based storage not suitable for high-scale deployments
